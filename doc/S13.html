NAME

MinG::S13 -- Stabler's (2013) parser.

INTERNAL CLASSES AND FUNCTIONS]

Priority: Objects in this class describe positions in a Queue.

bigger_than:(Priority $: Priority $other, *%_ --> Bool): A method that compares this object's priority with another Priority object's priority and returns true if this one's is smaller.
add_p:(Priority $: Int $n, *%_ --> Priority): Method that appends a number at the end of the priority. It returns a new Priority instead of changing itself automatically. Less efficient, but more useful for our purposes.
length:(Priority $: *%_ --> Int): Returns the length of the priority. In this implementation, priorities are sequences of numbers, and they are ordered lexicographically so that longer sequences have less priority than shorter ones.
bigger_pty:(Priority $a, Priority $b): Function used to compare two priorities. Uses Priority.bigger_than method internally. There aren't many reasons to use it, but if you can find one, go ahead.

Mover: Class for our movers: they represent the nodes we selected when we ran into a [MOVE, PLUS] feature and encountered a [MOVE, MINUS] feature as one of ROOT's children.

QueueItem: Objets of this class are items in a Queue. The only Queue we need to implement is the one for category predictions, so this is to be interpreted as an item in THAT queue.

bigger_than:(QueueItem $: QueueItem $other, *%_): This method is a wrapper around Priority.bigger_than so that it can be called more easily from a Queue object.
to_str:(QueueItem $: *%_ --> Str): Method that gets a QueueItem in string for to print while debugging.
deep_clone:(QueueItem $: *%_ --> QueueItem): Method that "deep-clones" a QueueItem. Actually, only @.movers needs to be deep-cloned (and that means simply copying the array. It doesn't get deeper than that), as the rest of the attributes don't get modified normally.

Queue: The Queue of category predictions.

ind_max:(Queue $: *%_ --> Int): With this method, we find out the index of the highest-priority item. Linear time.
max:(Queue $: *%_): Method that gets a reference to the highest-priority item. Linear time.
pop:(Queue $: *%_): Method that deletes the highest-priority item and returns it. Linear time.
push:(Queue $: QueueItem $new, *%_): Method that adds an element to the Queue. This runs in constant time.
elems:(Queue $: *%_ --> Int): Method that gets the amount of elements in the Queue. It's not reliable because we keep deleted items in the @.items array. We get the right result when there's 0 elements because Perl6 deletes pseudo-deleted elements from the end of the array (even if they were pseudo-deleted a long time ago). Note: I guess this could easily break with future implementations of Perl6.
deep_clone:(Queue $: *%_ --> Queue): Method that (partially) deep-clones a Queue. Some parts don't need to be deep-cloned, so we get references to that.

DerivTree: Class that represents derivation trees. As of now, they're just Nodes.

Derivation: Class that represents one derivation.

still_going:(Derivation $: *%_ --> Bool): Method that returns whether this derivation still needs more steps.
scan:(Derivation $: QueueItem $pred, Int $child_place, *%_ --> Derivation): See Stabler (2013)
merge1:(Derivation $: QueueItem $pred, Node @leaves, Node $selected, Node $selector, *%_ --> Derivation): See Stabler (2013)
merge2:(Derivation $: QueueItem $pred, Node @non_terms, Node $selected, Node $selector, *%_ --> Derivation): See Stabler (2013)
merge3:(Derivation $: QueueItem $pred, *%_ --> Derivation): See Stabler (2013)
merge4:(Derivation $: QueueItem $pred, *%_ --> Derivation): See Stabler (2013)
move1:(Derivation $: QueueItem $pred, *%_ --> Derivation): See Stabler (2013)
move2:(Derivation $: QueueItem $pred, *%_ --> Derivation): See Stabler (2013)
exps:(Derivation $: *%_ --> Array): Method that gets the expansions to be had in the next step. Check the code's comments for more details.
to_str:(Derivation $: *%_ --> Str): Method that beautifies a derivation. Mostly used for debugging purposes.

EXPORTED CLASSES AND FUNCTIONS

MinG::S13::Parser: Class that implements the parser per se. This is not where the magic happens, but it is where most of the external API is defined.

devq_to_str:(MinG::S13::Parser $: *%_): Method that gets a nice string representation of the @!devq.
parallel_run:(MinG::S13::Parser $: *%_): Method that runs one iteration of the parsing loop, running one step of each derivation in parallel. Gets all possible derivations.
procedural_run:(MinG::S13::Parser $: *%_ --> DerivTree): Method that runs one iteration of the parsing loop, running one step of one derivation only. No parallel computation.
parallel_parse:(MinG::S13::Parser $: *%_ --> Bool): Method that runs the main parsing loop using parallel_run. Gets all possible derivations.
procedural_parse:(MinG::S13::Parser $: *%_ --> Bool): Method that runs the main parseing loop using procedural_run. Stops after it finds the first derivation.
init:(MinG::S13::Parser $: MinG::Grammar $g, *%_): Method that initialises that parser to later parse several strings. Should be used instead of Parser.setup when a single grammar is going to be used several times.
parse_str:(MinG::S13::Parser $: Str $inp, ParseWay $do = ParseWay::PARALLEL, *%_): Method that parses a single string based on the grammar that was initialised using Parser.init()
setup:(MinG::S13::Parser $: MinG::Grammar $g, Str $inp, *%_): Method that sets up a parser with a certain grammar and a certain input (taken as a string for convenience, converted to lower case and an array as needed) and creates the first derivation.
parse_me:(MinG::S13::Parser $: MinG::Grammar $g, Str $inp, ParseWay $do = ParseWay::PARALLEL, *%_): Method to set the parser up, parse and spit the result. All in one go.
parse_and_spit:(MinG::Grammar $g, Str $inp, ParseWay $waytoparse = ParseWay::PARALLEL): This is a wrapper subroutine to save to avoid using the MinG::S13::Parser class directly.
